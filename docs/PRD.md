## 2. 背景与目标

### 2.1 背景

用户拥有一台飞牛 NAS，局域网内远程挂载了台式电脑的硬盘，NAS 本身也有多块机械硬盘和外接硬盘。里面存储了大量的视频和图片文件，目前缺乏统一管理工具。

### 2.2 目标

开发一个 **视频图片文件管理 Web 系统**，实现：

1. **视频管理** - 类抖音播放体验，快捷键操作
2. **图片管理** - 网格/列表浏览，缩略图预览
3. **统一去重** - 基于文件哈希识别重复文件
4. **批量清理** - 标记不喜欢，一键批量删除
5. **文件夹管理** - 保留层级结构或平铺显示

## 3. 用户场景

### 3.1 视频管理场景

**场景 1：刷视频放松**
> 作为用户，我希望像刷抖音一样浏览视频，我希望通过上下键切换视频，这样我可以轻松娱乐放松。

**场景 2：标记不喜欢**
> 作为用户，我希望标记不喜欢的视频，我希望这些视频不会再次出现，这样我可以专注于喜欢的内容。

**场景 3：批量清理**
> 作为用户，我希望一键删除所有标记为不喜欢的视频，我希望同时清理空文件夹，这样我可以快速释放存储空间。

### 3.2 图片管理场景

**场景 4：浏览图片收藏**
> 作为用户，我希望以网格方式浏览所有图片，我希望点击图片可以放大查看，这样我可以快速浏览和管理照片。

**场景 5：图片去重**
> 作为用户，我希望系统自动识别重复图片，我希望手动选择保留哪一张，这样我可以优化存储空间。

**场景 6：图片批量清理**
> 作为用户，我希望标记不喜欢的图片，我希望批量删除这些图片，这样我可以快速整理照片库。

### 3.3 文件夹管理场景

**场景 7：按文件夹浏览**
> 作为用户，我希望保留文件夹层级结构，我想要按文件夹导航，这样我可以按原来的目录快速定位文件。

**场景 8：全部平铺浏览**
> 作为用户，我希望把所有文件平铺显示，我想要在一个页面查看所有文件，这样我可以全局搜索和筛选。

## 6. 非功能需求

### 6.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面首次加载 | < 3 秒 |
| 视频启动时间 | < 2 秒 |
| 图片缩略图加载 | < 500ms |
| 列表翻页响应 | < 1 秒 |
| 批量删除 100 个 | < 10 秒 |

### 6.2 安全需求

- 用户密码加密存储（BCrypt）
- JWT Token 认证
- 敏感操作需登录验证
- 文件路径不暴露真实存储位置

### 6.3 兼容性需求

- 浏览器：Chrome, Firefox, Safari, Edge 最新两个版本
- 分辨率：1920x1080 及以上
- NAS：飞牛 OS (Linux)

### 6.4 数据安全

- SQLite 数据库每日备份
- 删除操作支持确认提示
- 不喜欢标记可撤销
- 文件删除仅移除记录，保留源文件（可选）

## 7. 风险评估

| 风险类型 | 风险描述 | 影响 | 应对措施 |
|---------|---------|------|---------|
| 技术风险 | 大规模文件扫描耗时过长 | 中 | 后台异步扫描，支持中断 |
| 技术风险 | 缩略图占用大量磁盘 | 中 | 定期清理未使用缩略图 |
| 数据风险 | 数据库损坏 | 高 | 定期自动备份 |
| 安全风险 | 未授权访问 | 高 | 强密码策略，JWT 过期 |
